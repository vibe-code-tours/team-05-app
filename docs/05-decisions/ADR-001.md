# ADR-001: Frontend Framework

- **Status:** Accepted
- **Date:** 2026-07-10

## Context

CrossMart is a cross-border e-commerce marketplace targeting Myanmar users, where mobile web is the primary access channel. The frontend must deliver:

- High-performance mobile-first experience (slow networks, budget Android devices)
- SEO for product listings (critical for marketplace discoverability)
- Server-side rendering (SSR) or static generation for fast initial loads
- Rich interactivity (cart, real-time cargo tracking, search)
- A mature ecosystem with UI component libraries (shadcn/ui integration)
- TypeScript-first development with strong type safety

The team needs a framework that minimizes configuration overhead while supporting both SSR and client-side hydration seamlessly.

## Decision

We chose **Next.js 15 (App Router)** as the frontend framework over the following alternatives:

| Alternative | Why Not |
|-------------|---------|
| **Vue 3 + Nuxt 3** | Smaller ecosystem for React-centric shadcn/ui components; team familiarity is stronger with React; fewer headless UI libraries with equivalent quality |
| **SvelteKit** | Promising performance but smaller ecosystem; fewer production-proven e-commerce patterns; limited component library ecosystem compared to React/shadcn |
| **Remix** | Overlapping capabilities with Next.js but smaller community; fewer deployment targets; less mature App Router equivalent |

### Key Reasons

1. **shadcn/ui compatibility:** shadcn/ui is built for React and TailwindCSS, matching our design system requirements perfectly.
2. **App Router:** Next.js 15 App Router provides RSC (React Server Components) for reduced client-side JavaScript, critical for low-end devices.
3. **Image optimization:** Built-in `next/image` handles responsive images, lazy loading, and format conversion (WebP/AVIF) for Myanmar's bandwidth constraints.
4. **ISR (Incremental Static Regeneration):** Product listing pages can be statically generated and revalidated, combining SEO with freshness.
5. **Ecosystem maturity:** Largest React framework community; extensive middleware, auth, and deployment integrations.
6. **Vercel + edge deployment:** Native integration with Vercel for zero-config deployment, edge functions for geo-relevant content serving.

## Consequences

### Positive

- **Best-in-class DX:** Hot reload, TypeScript support, file-based routing, and built-in API routes reduce boilerplate.
- **SEO out of the box:** SSR/ISR ensures product pages are crawlable by search engines without additional configuration.
- **Performance:** RSC reduces JavaScript bundle size, improving TTI on budget devices.
- **shadcn/ui integration:** Direct compatibility with our component library choice; copy-paste components that are fully customizable.
- **Scalability:** App Router supports parallel routes, intercepting routes, and streaming for complex marketplace UX patterns.

### Negative

- **Vendor lock-in risk:** Next.js-specific features (Image, App Router conventions) create some coupling; migration to another React framework would require refactoring.
- **Build complexity:** App Router is newer and has had breaking changes; occasional edge cases in RSC data fetching patterns.
- **Deployment flexibility:** While Vercel is optimal, self-hosting Next.js requires Node.js server infrastructure (unlike pure SSG frameworks).

### Neutral

- The team's existing React experience means a lower learning curve compared to Vue/Nuxt or Svelte alternatives.
- Next.js 15 is actively maintained with frequent releases, requiring periodic attention to breaking changes.
