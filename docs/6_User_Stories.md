# User Stories

This document captures the agile User Stories for the CrossMart platform.

## 1. Client (Buyer)
- **US-01:** As a Client, I want to register using my phone number so that I can quickly create an account.
- **US-02:** As a Client, I want to search for products by keyword so that I can easily find what I'm looking for.
- **US-03:** As a Client, I want to filter products by "In Stock" or "Cargo" so that I know how long delivery will take.
- **US-04:** As a Client, I want to view a Seller's rating and past successful orders so that I can trust them before purchasing.
- **US-05:** As a Client, I want to add items to a shopping cart so that I can buy multiple things at once.
- **US-06:** As a Client, I want to upload a screenshot of my KPay transfer during checkout so that I can prove I paid for the order.
- **US-07:** As a Client, I want to see a detailed timeline of my Cargo order so that I know exactly where my package is (e.g., Bangkok Warehouse vs Yangon).
- **US-08:** As a Client, I want to receive a notification when my order is "Out for Delivery" so I can be ready to receive it.
- **US-09:** As a Client, I want to rate the seller out of 5 stars after receiving my item so that others know about their service quality.

## 2. Sales Person (Seller)
- **US-10:** As a Seller, I want to apply for a verified account by uploading my ID so that clients trust my store.
- **US-11:** As a Seller, I want to list a new product with multiple images and variants (colors/sizes) so that clients have clear options.
- **US-12:** As a Seller, I want to list a "Promotion" item with a countdown timer so that I can gather orders while I am at a mall in Bangkok.
- **US-13:** As a Seller, I want my listings to be auto-approved (since I have a high rating) so that I can sell faster.
- **US-14:** As a Seller, I want to see a dashboard of all my pending orders so that I know what items I need to purchase and pack.
- **US-15:** As a Seller, I want to update an order's status to "Packed" and "Sent to Cargo" so that the Client is informed.
- **US-16:** As a Seller, I want to view the payment slip uploaded by the Client so that I can verify the funds in my bank account.

## 3. Admin
- **US-17:** As an Admin, I want to review new Seller applications so that I can prevent scammers from joining the platform.
- **US-18:** As an Admin, I want to review pending product listings from new sellers so that I can ensure they don't violate platform policies.
- **US-19:** As an Admin, I want to manage platform Categories and Brands so that the catalog remains organized.
- **US-20:** As an Admin, I want to view platform-wide sales reports so that I can track revenue and commission growth.
- **US-21:** As an Admin, I want to be able to ban a Seller or Client if they violate terms of service.
- **US-22:** As an Admin, I want to configure the rules for "Auto Approval" (e.g., minimum rating required) so that I can optimize the workload of manual reviews.

*(Note: This is a representative subset of the full 100+ user stories for the MVP phase. Additional granular stories will be tracked in Jira/GitHub Issues during Sprint Planning).*
